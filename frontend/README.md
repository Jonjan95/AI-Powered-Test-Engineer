# Frontend

The frontend is a minimal Next.js, TypeScript, and Tailwind CSS application. The homepage now provides the first project workflow: it lists projects from the backend, creates projects, selects a project, lists that project's user stories, and creates user stories.

The frontend API client reads `NEXT_PUBLIC_API_BASE_URL` to decide which backend to call. If the variable is not set, it defaults to `http://localhost:8080`.

Example local override:

```bash
NEXT_PUBLIC_API_BASE_URL=http://localhost:8080
```

Install dependencies and start the development server:

```bash
npm install
npm run dev
```

Open `http://localhost:3000`.

Run the local Playwright end-to-end test with:

```bash
npm run test:e2e
```

The Playwright config starts the Next.js dev server automatically and tests `http://localhost:3000`. The current smoke test mocks the project and user-story API responses in the browser so it can verify the frontend workflow reliably without requiring the Spring Boot backend to be running.

Useful Playwright commands:

```bash
npm run test:e2e:headed
npm run test:e2e:ui
npm run test:e2e:report
```

Playwright code generated by the backend is stored in PostgreSQL and is not automatically added to this test directory or executed.

See `../docs/setup.md` for first-time setup and `../docs/quick-start.md` for the daily workflow.
